# Security policy

## Reporting a vulnerability

Do not open a public issue, discussion, pull request, or social-media post for an exploitable vulnerability.

Use GitHub's private vulnerability reporting feature for this repository when it is enabled. If it is unavailable, contact the maintainers privately through a verified contact channel published on the official Olive Branch Network website. Include affected routes or commits, impact, reproduction steps, and any suggested mitigation. Do not include private keys, seed phrases, or unrelated personal information.

Allow the maintainers reasonable time to investigate and remediate the issue before disclosure. Do not access data or funds that do not belong to you, degrade production services, or perform testing against third-party accounts without permission.

## Scope reminders

This repository contains a frontend and server routes; deployed smart contracts are separate systems. Reports should clearly identify whether the issue affects this codebase, a deployed contract, or a third-party provider. Hiding frontend controls is never considered an authorization mechanism.
